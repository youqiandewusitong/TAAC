# 去年优秀方案总结

本文将去年整理出的几个优秀方案做一次统一归纳，便于后续写汇报、做答辩材料或二次迭代。整体上，这些方案都围绕生成式推荐展开，但各自侧重点不同：有的偏行为建模，有的偏结构增强，有的偏规模研究，有的偏系统创新。

## 一、总体结论

四个方案共同的方向是：

1. 用生成式建模替代传统两塔或粗排精排分离范式。
2. 用更强的序列建模、语义 ID、行为条件化和大规模负样本提升效果。
3. 在工程层面强调静态图、混合精度、ANN 检索和优化器分组，以兼顾训练效率与线上延迟。

如果只看“谁最像工业落地方案”，第一名最完整；如果看“谁最有理论研究价值”，第三名和技术创新奖更突出。

## 二、第一名：行为条件化多模态自回归生成推荐

### 核心定位

这是最完整的一体化方案，重点解决广告行为信号天然异质的问题，把曝光、点击、转化三类行为做显式解耦。

### 主要亮点

- 用 Qwen Backbone 作为主干，增强生成推荐的序列表达能力。
- 在每个位置做 Action Conditioning，通过 FiLM、Gated Fusion、Attention Biasing 三种机制联合调制表征。
- 引入三级时序特征、Fourier 周期编码和 Session 结构建模，补足传统位置编码对真实时间的缺失。
- 使用 RQ-KMeans 语义 ID，并通过 Random-k 正则防止语义退化。
- 训练上结合 Muon + AdamW、静态图 InfoNCE 和 Large Negative Banks，兼顾效果与稳定性。
- 推理阶段用用户向量 + Faiss ANN 做高效召回。

### 适用场景

适合强调“行为语义解耦 + 多模态 + 时序精建模”的推荐场景，工业完整度最高。

## 三、第二名：Encoder-Decoder + 图神经网络增强

### 核心定位

这个方案的思路是“先编码、再解码”，用更明确的结构分工来建模用户兴趣演化。

### 主要亮点

- Encoder 侧采用 Gated MLP 分别编码用户、物品和序列上下文。
- 用时序图神经网络补充高阶协同信号，缓解稀疏物品表征不足。
- Decoder 侧使用大容量 SASRec-style Transformer，显式生成下一兴趣向量。
- 语义 ID 采用 SVD 预降维 + RQ-KMeans，提升量化质量。
- 训练采用两阶段策略：先用曝光预训练，再用点击/转化微调。
- 推理后过滤历史交互物品，提升结果的新颖性。

### 适用场景

适合需要“结构清晰、模块职责明确、便于拆分实现”的方案设计。

## 四、第三名：生成推荐 Scaling Law 系统研究

### 核心定位

这个方案的重点不是复杂结构本身，而是系统研究“生成推荐里哪些 Scale 维度最有效”。

### 主要亮点

- 采用 Decoder-only Transformer 作为统一生成框架。
- 系统验证三个 Scaling 方向：负样本数量、模型容量、Item-ID Embedding 维度。
- 得出一个很重要的结论：在生成式推荐中，扩大量级足够大的负样本收益最明显。
- 结合 AMP 和 Static Graph Compilation，强调大模型训练效率。

### 适用场景

适合偏研究型汇报，特别适合讲“为什么要扩规模、扩哪里最值”。

## 五、技术创新奖：统一生成检索与排序

### 核心定位

这个方案强调把检索和排序统一到一个生成模型里，同时加入 MoE、碰撞解决、协同 Embedding 等系统级设计。

### 主要亮点

- 用一个 Decoder-only 模型同时预测语义 ID 和行为类型，实现检索与排序联合建模。
- 引入 FlashAttention、SwiGLU、RMSNorm、RoPE 和 DeepSeek-V3 风格 MoE，架构现代化程度高。
- 额外训练协同 Embedding，再与多模态 Embedding 融合后做语义 ID 量化。
- 设计了语义 ID 碰撞解决机制，提升码表利用率和长尾区分度。
- 结合多时间窗口热度统计，补充全局物品状态信号。

### 适用场景

适合强调“检索 + 排序一体化”和“系统级创新”的表达。

## 六、四个方案的横向比较

| 维度 | 第一名 | 第二名 | 第三名 | 技术创新奖 |
|---|---|---|---|---|
| 核心目标 | 行为语义彻底解耦 | 结构分层建模 | 研究 Scaling Law | 统一检索与排序 |
| 架构风格 | Qwen Decoder-only | Encoder-Decoder | Decoder-only | Decoder-only + MoE |
| 行为条件化 | 最强，三机制协同 | Decoder 侧注入 | 显式条件输入 | 统一目标中隐式体现 |
| 时序建模 | 最丰富 | 中等 | 基础时序 | 强调多窗口热度 |
| 语义 ID | RQ-KMeans + Random-k | SVD + RQ-KMeans | 未突出 | 协同 Emb + 碰撞解决 |
| 工程优化 | 最完整 | 中等 | AMP + 静态图 | MoE + Grouped GEMM |

## 七、最值得保留的共性经验

1. 生成式推荐比传统固定打分范式更适合统一建模复杂行为链路。
2. 行为类型不能当成普通特征直接拼接，最好做显式条件化。
3. 时间信息不能只靠位置编码，最好加真实时间、间隔和 Session 结构。
4. 语义 ID 对长尾物品和冷启动非常关键。
5. 大规模负样本是生成推荐里最重要的提升杠杆之一。
6. 训练和推理必须从一开始就考虑工程效率，否则很难真正落地。

## 八、最终推荐顺序

如果目标是写一个最完整、最像工业方案的总结，优先推荐第一名；如果目标是展示方法论清晰、模块边界明确的系统设计，第二名更合适；如果目标是强调实验结论和可扩展规律，第三名最适合；如果目标是突出创新点和系统整合能力，技术创新奖最亮眼。
