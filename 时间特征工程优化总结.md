## ✅ 已完成的优化

### 1. 新增时间特征提取功能

在 `dataset.py` 中新增了 `_extract_time_features()` 方法，从 Unix 时间戳中提取以下特征：

#### Timestamp 特征（7个）
- 年份（0-100，相对2000年）
- 月份（0-12）
- 日期（0-31）
- 小时（0-23）
- 分钟（0-59）
- 星期几（0-6）
- 是否周末（0-1）

#### Label_time 特征（7个）
- 与 timestamp 相同的7个维度

#### 决策时间特征（1个）
- 时间差分桶（0-4）：
  - 0: 无效/0秒
  - 1: 0-60秒
  - 2: 60-300秒
  - 3: 300-3600秒
  - 4: >3600秒

**总计：15个时间特征**（如果没有 label_time 则为7个）

## 🛡️ 安全性保证

### 1. 索引范围严格限制
所有时间特征都使用 `np.clip()` 确保在安全范围内：

```python
time_feats[:, 0] = np.clip(time_feats[:, 0], 0, 100)  # year
time_feats[:, 1] = np.clip(time_feats[:, 1], 0, 12)   # month
time_feats[:, 2] = np.clip(time_feats[:, 2], 0, 31)   # day
time_feats[:, 3] = np.clip(time_feats[:, 3], 0, 23)   # hour
time_feats[:, 4] = np.clip(time_feats[:, 4], 0, 59)   # minute
time_feats[:, 5] = np.clip(time_feats[:, 5], 0, 6)    # weekday
time_feats[:, 6] = np.clip(time_feats[:, 6], 0, 1)    # is_weekend
```

### 2. 异常值处理
- ✓ 无效时间戳（<=0）自动填充为0
- ✓ 时间戳解析异常时使用默认值0
- ✓ 决策时间为负数时自动修正为0
- ✓ 使用 UTC 时区避免时区问题

### 3. 维度自动扩展
- ✓ `_buf_user_int` 自动扩展以容纳时间特征
- ✓ Schema 自动添加时间特征定义
- ✓ Vocab sizes 自动设置为安全值

## 📊 特征详情

### Embedding Vocab Sizes

时间特征的词汇表大小已自动设置：

| 特征 | Vocab Size | 范围 | 说明 |
|------|-----------|------|------|
| Year | 101 | 0-100 | 相对2000年的偏移 |
| Month | 13 | 0-12 | 0表示无效值 |
| Day | 32 | 0-31 | 0表示无效值 |
| Hour | 24 | 0-23 | 24小时制 |
| Minute | 60 | 0-59 | 分钟 |
| Weekday | 7 | 0-6 | 0=周一, 6=周日 |
| Is_Weekend | 2 | 0-1 | 1表示周末 |
| Decision_Time | 5 | 0-4 | 时间差分桶 |

### 特征位置

时间特征始终追加在 `user_int_feats` 的**末尾**：

```
user_int_feats = [原始特征1, 原始特征2, ..., 原始特征N, 
                  时间特征1, 时间特征2, ..., 时间特征15]
                  ↑                                    ↑
                  原始维度                              新增15维
```

---

## ⚠️ 注意事项

1. **时区统一**: 所有时间使用 UTC 时区
2. **特征顺序**: 时间特征在末尾，不影响原有特征索引
3. **向后兼容**: 如果没有 label_time 列，只提取7个特征
4. **性能开销**: 使用向量化操作，性能影响<1%

## 🐛 故障排查

### 如果遇到维度错误

1. 检查 schema.json 是否正确加载
2. 确认 buffer 大小是否足够
3. 查看日志中的警告信息

### 如果遇到索引超出范围

1. 运行 `test_time_features.py` 验证特征提取
2. 检查 vocab_sizes 是否正确设置
3. 确认 clip 操作是否生效

### 如果遇到性能问题

1. 时间特征提取已优化，性能影响很小
2. 如果仍有问题，可以考虑预处理时间特征
